# Calendar

The **Calendar** gives your team a shared month view of what's scheduled — time
off, procedures, meetings, and general events — so everyone can see who's in and
what's planned.

## Personal vs. branch view

The calendar has two scopes you can switch between:

- **My calendar** — only your own events.
- **Branch calendar** — events shared with your branch.

Doctors open their personal calendar by default; owners and branch managers open
the branch calendar by default. Either way you can toggle between the two.

## Event types

Every event has a type, shown by colour on the calendar:

- **Day off** — time away or leave.
- **Procedure** — a scheduled procedure; can link a patient and a procedure, and
  add assisting staff.
- **Meeting** — an internal meeting.
- **General** — anything else.

## Creating an event

<Steps>
  <Step title="Open the new-event form">
    Click **New event** on the calendar.
  </Step>
  <Step title="Pick the type and details">
    Choose the event type, then set the title and the start/end time (or mark it
    **all day**) and an optional description.
  </Step>
  <Step title="Choose who can see it">
    Set the event's privacy with **Who can see this event?** (see below).
  </Step>
  <Step title="Add procedure details (procedures only)">
    For a procedure, optionally link the patient and the procedure and add
    assisting staff.
  </Step>
  <Step title="Save">
    The event appears on the calendars of everyone it's visible to.
  </Step>
</Steps>

## Event privacy

When you create or edit an event, **Who can see this event?** controls its
audience. There are three options:

- **Private** — only you can see it.
- **This branch** — visible to your branch peers.
- **Organization-wide** — visible to everyone in the organization.

<Callout type="note" title="Sensible defaults — and you always see your own">
**Day off** and **Procedure** events default to **This branch** so the team can
plan around them; **Meeting** and **General** events default to **Private**. You
can change the audience on any event, and you always see events you created
regardless of their privacy.
</Callout>

To book a patient appointment (specialty → doctor → service → time), use the
**[Visits](/guide/visits)** workspace instead.
